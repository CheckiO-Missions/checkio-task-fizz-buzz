The more pythonic way is to write as:

Ok, what numbers should I use instead of n, n1?

You should check divisibility from specific to general. Since, less numbers are evenly divisible by 15, then by 5 or 3, this case is more specific, so this checking should go first. 5 is not evenly divisible by 3, so the further order is not important.

How to complete conditions?

You need to add correct return to respective cases: strings "Fizz", "Buzz" and "Fizz Buzz".

In the other case (when num is not divisible by 15, 5, 3 evenly), the function should return num as string. If you need the string representation of a number, use str() built-in function.

So, how my final solution should look like?

Just wrap your code into function expression.

Here is the whole solution:

def checkio(num: int) -> str:

    if not num % 15:
        return "Fizz Buzz"
    elif not num % 5:
        return "Buzz"
    elif not num % 3:
        return "Fizz"
    else:  
        return str(num)

By the way, if your condition cases end with return statement, you usually do not need elif and else - if is enough: 

def checkio(num: int) -> str:

    if not num % 15:
        return "Fizz Buzz"
    if not num % 5:
        return "Buzz"
    if not num % 3:
        return "Fizz"
    return str(num)

The final answer is built from parts: if the string is multiplied by 0, it becomes an empty string "". 

def checkio(n: int) -> str:

    return 'Fizz'*(not n%3)+' '*(not n%15)+'Buzz'*(not n%5) or str(n)

In this solution division by 15 is avoided, but in case num is evenly divisible both by 5 and 3, it means, that it is also evenly divisible by 15. So both strings are present in the list.
