Ok, jak to zrobić?

Proszę sprawdzić podzielność od szczegółowej do ogólnej. Ponieważ mniej liczb jest parzyście podzielnych przez 15, a następnie przez 5 lub 3, ten przypadek jest bardziej szczegółowy, więc to sprawdzenie powinno być pierwsze. 5 nie jest parzyście podzielny przez 3, więc dalsza kolejność nie jest ważna.

Proszę o małą podpowiedź!

Proszę użyć tych samych narzędzi - operatora modulo (%) i instrukcji if-elif-else - aby sprawdzić różne przypadki dzielenia.

        ...
    elif not num % 5:
        ...
    elif not num % 3:

    else:

Jak powinno wyglądać moje ostateczne rozwiązanie?

Proszę po prostu dodać poprawny zwrot do odpowiednich przypadków.

Oto całe rozwiązanie:

def checkio(num: int) -> str:

    if not num % 15:
        return "Fizz Buzz"
    elif not num % 5:
        return "Buzz"
    elif not num % 3:
        return "Fizz"
    else:  
        return str(num)

Przy okazji, jeśli Państwa przypadki warunku kończą się instrukcją return, zazwyczaj nie potrzebują Państwo elif i else - wystarczy if: 

def checkio(num: int) -> str:

    if not num % 15:
        return "Fizz Buzz"
    if not num % 5:
        return "Buzz"
    if not num % 3:
        return "Fizz"
    return str(num)

Ostateczna odpowiedź jest zbudowana z części: jeśli ciąg zostanie pomnożony przez 0, stanie się pustym ciągiem "". 

def checkio(n: int) -> str:

    return 'Fizz'*(not n%3)+' '*(not n%15)+'Buzz'*(not n%5) or str(n)

W tym rozwiązaniu unikamy dzielenia przez 15, ale w przypadku, gdy num jest podzielny zarówno przez 5, jak i 3, oznacza to, że jest on również podzielny przez 15. Tak więc oba ciągi są obecne na liście. 

def checkio(num: int) -> str:
    
    res = []
    if not num % 3:
        res.append('Fizz')
    if not num % 5:
        res.append('Buzz')
    res = ' '.join(res) or str(num)

    return res
